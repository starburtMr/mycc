# ai-team（隔离区版）

## 适用场景

1. 任务跨前后端、测试、UI，多角色并行更高效。
2. 需要明确拆分与依赖管理，避免单代理上下文过载。

## 执行约束

1. 必须先读取任务真源：`tasks/active/<task>.md`。
2. 必须记录 `route_reason`，说明为何采用团队协作。
3. 不得直接修改 `.claude/`。
4. 只使用仓库内脚本，不依赖 `~/.claude/skills/*` 路径。

## 协作流程（最小可执行）

1. `PLAN`：Claude 产出拆分结果（子任务、依赖、验收标准）。
2. `ROUTE`：按子任务类型分配
   - 代码/后端/测试 -> `codex-agent`
   - UI/页面/样式 -> `gemini-agent`
3. `EXECUTE`：并行或串行执行子任务。
4. `REVIEW`：主代理整合结果并执行守护检查。
5. `UPDATE`：回写任务进展和反思。

## 输出要求

1. 子任务列表（含负责人、依赖、状态）。
2. 关键交付物路径。
3. 风险与回滚路径。
4. 下一步动作（可直接执行）。
