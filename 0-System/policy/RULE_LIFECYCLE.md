# 规则生命周期（MVP）

## 阶段

1. `staging`：新规则先进入暂存区。
2. `stable`：连续 3 次会话验证通过后晋级。
3. `archive`：验证失败或被替代后归档。

## 晋级条件

- 规则可触发、可验证。
- 至少 3 次会话 `result=pass`。
- 无硬评估回归（测试失败、巡检失败）。

## 回滚条件

- 任一会话出现硬评估失败且可归因于该规则。
- 连续 2 次软评分下降（style_score 或可读性评分下降）。

## 执行原则

- 先回滚再优化，避免污染稳定规则。
- 所有动作必须写入 `decision-log.md`。
