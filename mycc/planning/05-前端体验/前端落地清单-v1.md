## 1. 路由与页面

1. `/chat`
- 聊天主入口。
- 默认模式：`dispatch_mode=auto`。
- 仅显示对话 + Task 摘要卡。

2. `/tasks`
- 任务列表页（搜索、筛选、排序）。

3. `/tasks/[taskId]`
- 任务详情页（支持 `run_id` 切换）。
- 包含：Timeline / Output / Artifacts / Debug。

4. `/memory`
- 记忆浏览页（Working / Facts / Decisions）。

5. `/skills`
- 技能治理页（状态、blast_radius、最近使用/验证）。

6. `/system`
- 系统状态页（连接模式、健康状态、存储占用）。

## 3. 状态与数据边界

1. TanStack Query（服务端状态）
- `tasks list`
- `task snapshot`
- `skills list`
- `system status`

2. Zustand（UI 状态）
- `activeSessionId`
- `drawerTab`
- `filters/sorts`
- `composerDraft`
- `dispatchMode`

3. SSE 事件处理
- 输入：`/v1/tasks/:taskId/events?after=<seq>`
- 规则：按 `seq` 去重，缺口触发回放。
- Chat 页仅消费：`assistant_delta/task_started/task_status/task_result`。

## 4. API 映射表（前端实现）

1. 聊天发送
- `POST /v1/chat/send`
- 请求：`message, dispatch_mode, session_id`
- 响应：`task_created` + 可选 `task_id/run_id`

2. 任务列表
- `GET /v1/tasks`

3. 任务详情
- `GET /v1/tasks/:taskId`

4. 任务事件流
- `GET /v1/tasks/:taskId/events?after=<seq>`

5. 任务控制
- `POST /v1/tasks/:taskId/cancel`
- `POST /v1/tasks/:taskId/retry`
- `POST /v1/tasks/:taskId/resume`

6. 系统状态
- `GET /v1/system/status`

## 5. UI 与设计系统落实项

1. Tailwind token 接入（Dark First）
- `bg-base/bg-panel/bg-elevated`
- `text-primary/text-secondary`
- `primary/success/warning/error/info`

2. 字体
- UI：`Inter`
- 代码与事件：`JetBrains Mono`

3. 图标
- 统一 `lucide-react`

4. 动效
- 统一 `150ms ease-out`

## 6. 开发顺序（建议）

1. P1：`/chat` + `ChatComposer` + `dispatch_mode` + Task 摘要卡。
2. P2：`/tasks` + `/tasks/[taskId]` 基础骨架。
3. P3：SSE 回放与 Debug Panel。
4. P4：`/memory`、`/skills`、`/system` 页面。
5. P5：视觉 Token 全量替换与一致性收敛。

## 7. 验收清单（可测试）

1. `auto` 普通问答不创建任务。
2. `force_task` 必定创建任务并出现 Task 摘要卡。
3. `chat_only` 不创建任务。
4. Chat 页不出现 tool/routing/verification 细节。
5. Task 页可切 run 并支持 replay。
6. 断线重连后事件按 `seq` 连续，无重复展示。
7. 状态色与枚举在各页面一致。

## 8. 回滚方案

1. 若新组件引发渲染问题，先回退到 `demo` 骨架版布局。
2. 保留 API 契约不回退，仅回退 UI 绑定层。
3. SSE 逻辑问题优先降级为“仅 snapshot 刷新”，保持可用。
