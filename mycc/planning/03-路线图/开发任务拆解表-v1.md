# MyCC 开发任务拆解表（v1）

> 目标：将 v2 规划转换为可执行任务，支持前后端并行推进。

## 1. 迭代范围

- Iteration A：契约与后端主干
- Iteration B：前端入口与流式联调
- Iteration C：规则闸门与稳定性补强

## 2. 任务清单（按优先级）

## A. 契约与基线（必须先完成）

1. A1 技术栈冻结评审通过
- 输入：`00-基线/技术栈冻结-v1.1.md`
- 产出：冻结版本清单（可执行）
- 负责人：架构
- 验收：前后端环境一致可运行

2. A2 API 契约定稿
- 输入：`02-契约/API契约草案-v1.1.md`
- 产出：v1.1 定稿版（字段与错误码冻结）
- 负责人：后端 + 前端
- 验收：无歧义字段，前后端签字确认

3. A3 事件字典定稿
- 输入：`02-契约/事件字典-v1.1.md`
- 产出：事件类型、payload、seq 规则冻结
- 负责人：后端 + 前端
- 验收：前端可按字典渲染，不需要猜字段

## B. 后端主干（并行起点）

4. B1 后端骨架初始化
- 范围：`mycc/backend`
- 产出：可启动服务 + health 接口
- 负责人：后端
- 验收：`GET /v1/system/status` 返回正常

5. B2 run 模型与事件账本
- 产出：`tasks/task_runs/task_events/artifacts` 最小落库
- 负责人：后端
- 验收：任务执行全链路可回放

6. B3 SSE 回放能力
- 产出：`Last-Event-ID` / `after` 支持
- 负责人：后端
- 验收：断线重连不丢事件

7. B4 任务控制端点
- 产出：`cancel/retry/resume`
- 负责人：后端
- 验收：多 run 可追溯，状态一致

## C. 前端入口（并行推进）

8. C1 前端骨架初始化
- 范围：`mycc/frontend`
- 产出：Next.js + Tailwind + 基础布局
- 负责人：前端
- 验收：本地页面可运行

9. C2 聊天入口 + 任务入口统一
- 产出：聊天页可发起任务并显示流式结果
- 负责人：前端
- 验收：从输入到任务创建成功

10. C3 事件面板与调试抽屉
- 产出：状态/内容/审计三类事件展示
- 负责人：前端
- 验收：显示 route_reason、verification、memory_write

## D. 系统规则闸门（核心差异能力）

11. D1 EvoMap 前置闸门
- 产出：任务执行前必须先检索，失败阻断
- 负责人：后端
- 验收：可追踪预检事件

12. D2 双引擎路由记录
- 产出：每次模型选择写入 `model_routed`
- 负责人：后端
- 验收：`route_reason` 完整可查

13. D3 skill registry 强校验
- 产出：只允许 registry 通过的技能执行
- 负责人：后端
- 验收：未注册技能调用被拒绝并记录

## E. 稳定性与安全（P1）

14. E1 配对强化
- 产出：短码 TTL + challenge + 限流审计
- 负责人：后端
- 验收：暴力猜解被阻断

15. E2 local/lan/tunnel 模式
- 产出：三模式切换与权限策略
- 负责人：后端
- 验收：默认 local，tunnel 按需可用

16. E3 联调回归
- 产出：关键链路回归清单
- 负责人：前后端
- 验收：创建任务、断线重连、重试恢复全通过

## 3. 里程碑定义

1. M1（可联调）：A1-A3 + B1 + C1
2. M2（可演示）：B2-B4 + C2-C3
3. M3（可内测）：D1-D3 + E1
4. M4（可稳定使用）：E2-E3

## 4. 风险与缓解

1. 契约漂移风险
- 缓解：先冻结契约，再编码

2. SSE 不稳定风险
- 缓解：先落库后推流 + cursor 回放

3. 模型路由不可解释风险
- 缓解：强制 route_reason 事件化

4. 远程访问安全风险
- 缓解：默认 local，tunnel 非默认
