# SSE 可靠性测试用例（v1）

## 1. 验收目标

1. 断线后可恢复。
2. 不丢事件。
3. 不重复渲染。
4. 不乱序。

## 2. 核心用例

1. 断线重连回放
- 前置：已收到 `seq=120`。
- 操作：断开连接，重连时 `after=120`。
- 预期：只收到 `seq>120` 事件。

2. 重复事件去重
- 前置：客户端重复收到同一 `seq`。
- 预期：前端仅渲染一次。

3. 顺序一致性
- 前置：run 持续推流。
- 预期：`seq` 严格递增，无回退。

4. 推流失败回放
- 前置：服务端推流阶段网络波动。
- 预期：事件已落库，重连后补齐。

5. 长连接心跳
- 前置：长时间无业务事件。
- 预期：周期性 `heartbeat` 保持连接活性。

## 3. 结果记录

1. 每个用例记录 `request_id/task_id/run_id`。
2. 失败用例附 `evidence_ref`。
