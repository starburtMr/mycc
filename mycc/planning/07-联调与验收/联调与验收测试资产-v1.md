# 联调与验收测试资产（v1）

> 目标：把 Phase2/3 验收口径变成可执行测试资产，减少“口径有了但没人可复现”。

## 1. 测试资产分类

1. Replay Fixtures（协议与前端联调）
- 固定事件流 `*.jsonl`
- 不依赖真实模型与外网

2. SSE 可靠性用例（连接稳定性）
- 断线重连回放
- 重复事件去重
- seq 单调性
- 终态事件完整性

3. 端到端场景集（业务闭环）
- chat_only
- auto（不建任务）
- auto（建任务）
- force_task
- retry/resume

## 2. 目录约定

```text
tests/
  fixtures/
    normal_success.jsonl
    retryable_failure.jsonl
    stream_reset.jsonl
  sse/
    reconnect.spec.md
    dedup.spec.md
    seq_order.spec.md
  e2e/
    chat_dispatch_mode.spec.md
    task_retry_resume.spec.md
```

## 3. 关键验收项

1. `dispatch_mode` 验收：
- `chat_only` 永不建任务
- `auto` 普通问答不建任务
- `force_task` 必建任务

2. SSE 验收：
- `after=seq` 回放无缺口
- 重复 seq 只渲染一次
- `task_completed/task_failed` 至少一个终态事件存在

3. Debug 验收：
- `model_routed.route_reason` 可查
- `verification` 与 `memory_write` 在任务页可见

## 4. 执行节奏

1. CI 必跑：fixtures 校验 + SSE 核心用例。
2. 每次契约变更：重放一次全部 fixtures。
3. 每次版本发布前：跑完整 e2e 场景集。

## 5. 失败处理

1. fixture/schema 不匹配：阻断合并。
2. SSE 去重失败：阻断发布。
3. 终态事件缺失：阻断发布并回归排查。
