# SQLite 迁移与备份恢复（v1）

## 1. 目标

1. 表结构可持续演进。
2. 数据可备份可恢复。

## 2. 迁移规范

1. 使用 `schema_migrations` 记录版本。
2. 每次迁移必须可重复执行（幂等）。
3. 迁移脚本必须附回滚说明。

## 3. 备份范围

1. `memory/memory.db`
2. `mycc/backend/data/*.db`
3. `tasks/<task_id>/runs/*`（含 manifest）

## 4. 恢复流程

1. 恢复数据库快照。
2. 运行一致性检查（schema + 关键索引）。
3. 抽样回放 run 事件验证可读。

## 5. 演练频率

1. 日常备份：每日。
2. 恢复演练：每月。
