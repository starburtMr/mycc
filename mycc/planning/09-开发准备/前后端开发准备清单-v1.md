1. 契约基线：`02-契约/API契约草案-v1.2.md`

5. 规范基线：`0-System/rules/ENGINEERING_CONSTITUTION.md`

## 2. 后端开工前检查

1. 初始化工程骨架（Fastify + TS + migration）
2. `task_events` 回放链路优先打通（先落库后推流）
3. `chat/send` 的 `dispatch_mode` 三态落实
4. adapter 接口统一（Claude/Codex 同一输入输出协议）
5. 最小验收接口可用：
- `GET /v1/system/status`
- `POST /v1/chat/send`
- `GET /v1/tasks/:taskId/events?after=`

## 3. 前端开工前检查

1. 路由骨架：
- `/chat`
- `/tasks`
- `/tasks/[taskId]`
- `/memory`
- `/skills`
- `/system`
2. Chat/Task 边界：
- Chat 页仅对话 + Task 摘要卡
- Task 页展示执行与调试细节
3. SSE 回放与去重机制按 `seq` 实现
4. token 与组件按 Agent IDE 规范落地

## 4. 并行协作规则（蜂巢模式）

1. 子任务必须声明：
- scope
- non_goals
- allowed_files
- tests
- rollback
- commit_type
2. 合并前必须通过策略检查：
- `0-System/rules/engineering-policy.yaml`
3. 任一子任务不得修改未授权文件

## 5. 本阶段完成定义（DoD）

1. 前后端均可本地启动
2. 聊天可触发任务化并显示摘要卡
3. 任务页可订阅/回放事件流
4. 至少一条 Claude/Codex 路由链路可跑通
5. 关键路径有可执行验证命令
