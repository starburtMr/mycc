# 蜂巢并行执行协议（v1）

> 目标：让多代理并行开发可控、可回收、可审计。

## 1. 任务拆分粒度

1. 一个子代理只负责一个清晰功能点
2. 单步建议：
- 改动文件 <= 5
- 净新增 <= 200 行
3. 超限必须拆分并说明原因

## 2. 子代理任务卡模板（强制）

```yaml
task_id: HIVE-XXX
scope: <只写本次做什么>
non_goals: <明确不做什么>
allowed_files:
  - <path>
tests:
  - <command>
rollback: <git revert 或回退步骤>
commit_type: feat|fix|refactor|test|docs|chore
```

## 3. 执行顺序

1. 主代理分发任务卡
2. 子代理并行提交候选改动
3. 主代理统一做：
- 契约一致性检查
- 冲突合并
- 端到端回归
- 最终提交

## 4. 冲突处理

1. 契约冲突优先级：
- API 契约 > 事件字典 > 页面实现
2. 文件冲突由主代理裁决，子代理不得强行覆盖
3. 未经授权修改 `allowed_files` 之外文件，一律退回

## 5. 合并闸门

1. 必须通过：
- `engineering-policy.yaml` 规则检查
- 对应测试命令
- 回滚路径验证
2. 任一闸门失败，不得合并

## 6. 交付格式

每个子代理必须输出：
1. 改动摘要
2. 风险点
3. 验证结果
4. 回滚方式
5. 待主代理决策项（如有）
