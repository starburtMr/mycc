# MyCC 技术栈冻结（v1.2）

> 在 v1.1 基础上补强“契约落地闭环、版本锁定、原子写与流式一致性”。

## 1. 版本锁定策略

1. Node.js 锁定 22.x LTS，使用 `engines` + `.nvmrc`（或 Volta）双保险。
2. Next.js/Fastify/TypeScript 锁定到已验证 patch（不使用宽松范围）。
3. 所有依赖变更必须伴随 lockfile 更新与回归结果。

## 2. 契约落地闭环（必须）

1. `packages/contracts`：zod schema + 错误码枚举。
2. `packages/client`：由 contracts 生成 typed client（REST + SSE 参数类型）。
3. 前后端禁止手写重复 DTO。

## 3. 文件落盘规则

1. 产物目录：`tasks/<task_id>/runs/<run_id>/`。
2. 原子写：先写 `*.tmp`，`fsync` 后 `rename`。
3. 每次写入同时更新 `manifest.json`（path/hash/type/created_at）。

真源分层说明：
1. 事件真源：`task_events`（SQLite，可回放、可重建）。
2. 文件真源：`tasks/<task_id>/runs/<run_id>/`（产物与 manifest）。
3. `task_snapshot` 仅为缓存层，可由事件真源重建。

## 4. SSE 实现基线

1. SSE 必须统一通过单一 writer 封装，不允许路由各自手写。
2. 必须支持心跳、断线回放、seq 去重。
3. 事件先落库再推流。

## 5. 安全默认策略

1. `local`：仅 `127.0.0.1`，允许无鉴权。
2. `lan/tunnel`：强制 session 鉴权（cookie 优先）。
3. query token 仅作为兼容兜底，需 TTL 与撤销机制。
