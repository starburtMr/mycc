# MyCC 技术栈冻结（v1.1）

> 范围：前后端 MVP（P0-P1）

## 1. 目标与原则

1. 先稳定可运行，再做功能扩展。
2. 先契约后编码，前后端共享类型。
3. 默认本地模式，按需开启远程能力。

## 2. 版本冻结

## 2.1 运行时

- Node.js: 22.x LTS（全员统一）
- npm: 跟随 Node 22 默认版本
- TypeScript: 5.x

## 2.2 前端

- Next.js: 16.x（锁定到已验证 patch）
- React: 跟随 Next.js 官方要求
- 样式: Tailwind CSS（强制）
- UI 组件: shadcn/ui（可选，优先复用）
- 状态管理:
  - 服务端状态：TanStack Query
  - UI 状态：zustand

## 2.3 后端

- 框架: Fastify 5.x
- 校验: zod
- 数据库: SQLite（better-sqlite3）
- 日志: pino
- 流式: SSE（Fastify + 原生事件流）

## 2.4 数据与存储

- 任务真源：`tasks/`
- 结构化记忆：`memory/memory.db`
- 后端运行数据：`mycc/backend/data/`
- 产物目录：`tasks/<task_id>/runs/<run_id>/`

## 4. 强制工程约束

1. 执行前置：必须先进行 EvoMap 经验检索。
2. EvoMap 仅只读：禁止注册节点、heartbeat、接任务。
3. 每次模型路由必须记录 `route_reason`。
4. `tasks/` 为任务状态唯一真源，后端不得绕过。
5. Skill 调用必须来自 `skills-core/skill-registry.yaml` 有效项。

## 5. API 与类型策略

1. 后端以 zod schema 为单一契约源。
2. 自动生成 OpenAPI。
3. 前端使用共享类型或自动生成 typed client。
4. 禁止手写重复 DTO。

## 6. 流式与可靠性基线

1. SSE 必须支持心跳。
2. SSE 必须支持 `Last-Event-ID` 或 `after` 回放。
3. 事件必须带 `seq`（单调递增）用于幂等去重。
4. 任务必须支持 `cancel/retry/resume`。

## 7. 安全与连接模式

- local：默认，仅监听 `127.0.0.1`
- lan：局域网访问，需要鉴权
- tunnel：按需启用，默认关闭

配对策略（MVP）：
1. 短码限时（60-120 秒）。
2. 限流与失败锁定。
3. 全链路审计事件落盘。

## 8. 暂不引入

1. 重型微服务拆分。
2. 与目标无关的新依赖。
3. 默认启用 tunnel。
4. 原生移动端双栈。
