# MyCC 事件字典（v1.2）

## 1. 字段稳定性规则

1. 事件信封固定字段：`event_id, seq, task_id, run_id, ts, type, payload, meta`。
2. `seq` 为 run 内严格递增整数。
3. `meta.request_id` 必填。

## 2. 新增事件

1. `run_checkpointed`：写入阶段检查点（供 resume）。
2. `stream_reset`：流重置或 run 切换提示。
3. `warning`：非致命告警。

## 3. 既有事件补充

## 3.1 `tool_io`

payload：
- `tool` 必填
- `input` 可选
- `output_summary` 可选
- `output_ref` 可选（大结果落盘引用）

## 3.2 `task_failed`

payload 最小必填：
```json
{
  "status": "failed",
  "code": "INTERNAL_ERROR",
  "message": "执行超时",
  "retryable": true,
  "stage": "EXECUTE",
  "evidence_ref": "evt_120"
}
```

## 4. 前端消费新增规则

1. 遇到 `stream_reset` 时前端清理临时增量缓冲。
2. `warning` 在 UI 以黄色提示，不中断主流程。
3. `task_failed.retryable=true` 时展示“重试”按钮。

## 5. 后端实现新增规则

1. 同一 run 的 `seq` 分配与事件写入必须在事务内完成。
2. 推流异常不回滚事件写入。
