# MyCC 事件字典（v1.2）

## 1. 字段稳定性规则

1. 事件信封固定字段：`event_id, seq, task_id, run_id, ts, type, payload, meta`。
2. `seq` 为 run 内严格递增整数。
3. `meta.request_id` 必填。

## 2. 新增事件

1. `run_checkpointed`：写入阶段检查点（供 resume）。
2. `stream_reset`：流重置或 run 切换提示。
3. `warning`：非致命告警。
4. `task_started`：聊天请求被任务化后发出（用于聊天摘要卡）。
5. `task_status`：任务状态摘要变更（用于聊天摘要卡刷新）。
6. `task_result`：任务完成/失败后的结果摘要（用于聊天摘要卡收口）。

字段约束：
1. `task_started.payload` 最小字段：
```json
{"status":"running","stage":"PLAN","progress":10}
```
2. `task_status.payload` 最小字段：
```json
{"status":"running","stage":"EXECUTE","progress":45}
```
3. `task_result.payload` 最小字段：
```json
{"status":"completed","summary":"xxx","artifacts_count":3}
```
4. `task_result` 失败场景必须包含：
```json
{"status":"failed","code":"INTERNAL_ERROR","message":"xxx","retryable":true}
```

## 3. 既有事件补充

## 3.1 `tool_io`

payload：
- `tool` 必填
- `input` 可选
- `output_summary` 可选
- `output_ref` 可选（大结果落盘引用）

## 3.2 `task_failed`

payload 最小必填：
```json
{
  "status": "failed",
  "code": "INTERNAL_ERROR",
  "message": "执行超时",
  "retryable": true,
  "stage": "EXECUTE",
  "evidence_ref": "evt_120"
}
```

## 4. 前端消费新增规则

1. 遇到 `stream_reset` 时前端清理临时增量缓冲。
2. `warning` 在 UI 以黄色提示，不中断主流程。
3. `task_failed.retryable=true` 时展示“重试”按钮。
4. 聊天页只消费 `assistant_delta/task_started/task_status/task_result`。
5. `tool_io/model_routed/verification/memory_write` 仅在任务页 Debug 展示。

## 5. 后端实现新增规则

1. 同一 run 的 `seq` 分配与事件写入必须在事务内完成。
2. 推流异常不回滚事件写入。
3. `model_routed` 必须早于首个 `tool_io` 或 `assistant_delta` 写入。
4. `task_started` 仅在 `task_created=true` 时产生；`chat_only` 禁止产生。
