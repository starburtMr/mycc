# MyCC 事件字典（v1.1）

> 用途：规范 SSE 事件语义，确保前后端渲染一致、可回放、可审计。

## 1. 事件信封（统一结构）

```json
{
  "event_id": "evt_xxx",
  "seq": 1,
  "task_id": "tsk_xxx",
  "run_id": "run_xxx",
  "ts": "2026-03-05T14:30:00Z",
  "type": "stage_changed",
  "payload": {},
  "meta": {
    "request_id": "req_xxx",
    "source": "orchestrator"
  }
}
```

字段说明：
1. `event_id`：全局唯一。
2. `seq`：run 内单调递增，客户端按此去重。
3. `task_id/run_id`：定位任务与运行实例。
4. `type`：事件类型。
5. `payload`：事件业务数据。
6. `meta`：诊断辅助字段。

## 2. 状态类事件

## 2.1 `run_created`

触发：新 run 创建。

payload：
```json
{ "status": "queued" }
```

## 2.2 `status_changed`

触发：任务状态变化。

payload：
```json
{ "from": "queued", "to": "running", "reason": "调度开始" }
```

## 2.3 `stage_changed`

触发：编排阶段切换。

payload：
```json
{ "from": "PLAN", "to": "EXECUTE" }
```

## 2.4 `progress_updated`

触发：进度更新。

payload：
```json
{ "progress": 45 }
```

## 3. 内容类事件

## 3.1 `assistant_delta`

触发：模型增量输出。

payload：
```json
{ "channel": "final", "text": "已完成第一步..." }
```

## 3.2 `tool_io`

触发：工具调用摘要。

payload：
```json
{
  "tool": "web_search",
  "input": "备份恢复系统优化",
  "output_summary": "命中 3 条相关经验"
}
```

## 3.3 `artifact_emitted`

触发：生成可落盘产物。

payload：
```json
{
  "artifact_id": "art_xxx",
  "path": "tasks/tsk_xxx/runs/run_xxx/plan.md",
  "type": "markdown",
  "hash": "sha256:..."
}
```

## 4. 审计类事件

## 4.1 `model_routed`

触发：每次模型路由决策。

payload：
```json
{
  "engine": "claude",
  "candidates": ["claude", "codex"],
  "route_reason": "当前阶段为方案规划，优先 Claude"
}
```

## 4.2 `verification`

触发：验证步骤输出。

payload：
```json
{ "passed": true, "summary": "基础检查通过" }
```

## 4.3 `memory_write`

触发：记忆写入成功。

payload：
```json
{
  "namespace": "task",
  "layer": "working",
  "record_id": "mem_xxx",
  "evidence_ref": "evt_28"
}
```

## 5. 生命周期事件

## 5.1 `task_completed`

payload：
```json
{ "status": "completed", "summary": "任务完成" }
```

## 5.2 `task_failed`

payload：
```json
{ "status": "failed", "error_code": "INTERNAL_ERROR", "message": "执行超时" }
```

## 5.3 `heartbeat`

触发：长连接保活。

payload：
```json
{ "alive": true }
```

## 6. 前端消费规则

1. 按 `seq` 去重。
2. 断线后以 `Last-Event-ID` 或 `after` 补齐缺口。
3. 主视图展示内容类事件，侧栏展示审计类事件。
4. `task_failed`、`task_completed` 视为 run 终态。

## 7. 后端实现规则

1. 每个事件先落库再推流（防丢失）。
2. `seq` 必须连续，不允许回退。
3. 推流失败不影响落库，客户端可回放恢复。
4. 必须周期性发送 `heartbeat`。
