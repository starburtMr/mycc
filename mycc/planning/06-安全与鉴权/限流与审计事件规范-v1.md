# 限流与审计事件规范（v1）

## 1. 限流对象

1. 配对接口：`POST /v1/chat/pair`
2. 聊天入口：`POST /v1/chat/send`
3. 任务入口：`POST /v1/tasks`
4. SSE 建连：`GET /v1/tasks/:taskId/events`

## 2. 规则建议（MVP）

1. 配对失败超过阈值后短期锁定。
2. 同一来源短时间高频请求触发 429。
3. SSE 同时连接数设上限。

## 3. 审计事件

1. `auth_pair_attempt`
2. `auth_pair_success`
3. `auth_pair_blocked`
4. `rate_limit_hit`
5. `sse_auth_failed`

## 4. 审计字段

1. `request_id`
2. `source_ip`（可脱敏）
3. `session_id`（如有）
4. `task_id/run_id`（如有）
5. `reason`
6. `ts`

## 5. 错误返回

1. 命中限流统一返回 `RATE_LIMITED`。
2. 响应中不包含敏感内部细节。
