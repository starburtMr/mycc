# 鉴权与三模式安全策略（v1）

> 目标：把 `local/lan/tunnel` 从“原则”落到“可执行 checklist”。

## 1. 模式定义

1. `local`
- 仅监听 `127.0.0.1`。
- 可允许无鉴权（开发模式）。
- 禁止对外网卡暴露。

2. `lan`
- 必须 session/cookie 鉴权。
- 必须限流与审计。
- 必须开启 CSRF 防护。

3. `tunnel`
- 必须 session/cookie 鉴权。
- 必须限流与审计。
- 更短 session TTL。
- 必须 HTTPS 终端加密。

## 2. 会话与令牌策略

1. 优先 cookie session（`HttpOnly + Secure + SameSite=Lax/Strict`）。
2. query token 仅兼容兜底（SSE 特殊场景），必须：
- 短 TTL（建议 <= 5 分钟）
- 可撤销
- 日志脱敏

3. 刷新策略：
- access/session 过期后通过受控刷新流程续期。
- 刷新失败触发重新配对/登录。

## 3. 配对协议（防重放）

1. 生成 `pair_code + challenge + nonce`。
2. TTL 60~120 秒。
3. 客户端提交 `pair_code + challenge + nonce + ts`。
4. 服务端校验：
- nonce 一次性
- ts 在窗口期内
- 签名/校验通过
5. 成功后颁发 session，立即作废 challenge/nonce。

## 4. SSE 鉴权策略

1. 优先同域 cookie 自动鉴权。
2. 若使用 query token：
- token 必须短期有效
- token 与 `task_id/run_id` 绑定
- 连接关闭后主动失效（可选）
3. SSE 请求必须记录 `request_id/task_id/run_id` 审计字段。

## 5. 限流策略（最小）

1. `POST /v1/chat/pair`：按 IP + 设备指纹限流。
2. `POST /v1/chat/send`：按 session 限流。
3. `GET /v1/tasks/:taskId/events`：并发连接上限。
4. 连续失败触发退避与短时封禁。

## 6. 审计与脱敏

1. 审计最小字段：
- `request_id`
- `session_id`
- `task_id/run_id`（如有）
- `mode(local/lan/tunnel)`
- `result(success/fail)`

2. 日志脱敏：
- token/cookie/pair_code 不得明文落日志。
- 错误响应禁止回显敏感字段。

## 7. 安全验收清单

1. lan/tunnel 在无 session 时访问受限接口返回 401/403。
2. query token 过期后不可复用。
3. pair_code 重放失败且触发审计事件。
4. SSE 在鉴权失效后主动断连并可恢复登录。
