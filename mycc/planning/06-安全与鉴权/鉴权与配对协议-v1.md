# 鉴权与配对协议（v1）

## 1. 目标

1. 在 `local/lan/tunnel` 三种连接模式下给出可执行且可审计的鉴权方案。
2. 与 `chat/send 统一入 run`、`events?after=seq` 设计保持一致。

## 2. 模式策略（默认写死）

1. `local`
- 仅绑定 `127.0.0.1`。
- 可允许无鉴权。

2. `lan`
- 必须登录态（session/cookie）。
- 必须启用限流与审计事件。

3. `tunnel`
- 必须登录态（session/cookie）。
- 必须启用限流与审计事件。
- 会话过期时间更短。

## 3. 配对状态机

1. 生成 `pair_code`（短码） + `challenge`。
2. 设置 TTL（60-120 秒）。
3. 客户端提交 `pair_code + challenge`。
4. 校验通过后颁发 session（优先 cookie）。
5. challenge 一次性作废。

## 4. SSE 鉴权

1. 优先 cookie session。
2. 仅兼容场景允许 query token。
3. query token 必须有 TTL 与撤销机制。
4. 日志必须脱敏 token。

## 5. 安全要求

1. 配对失败触发限流。
2. 配对成功/失败都要产生日志与审计事件。
3. 禁止在错误响应中回显敏感凭据。
