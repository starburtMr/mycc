# Planning 索引

> 规则：规划内容按主题分子目录管理，不在根目录堆叠。

## 目录约定

- `00-基线/`：技术栈冻结、工程约束、版本策略
- `01-架构/`：架构设计、重设计方案、模块边界
- `02-契约/`：API 契约、事件字典、错误码规范
- `03-路线图/`：分期计划、任务拆解、里程碑

## 子目录扩展规则

当规划进入新阶段或出现明显新主题时，新增子目录，命名格式：

`NN-主题名`

示例：
- `04-安全与鉴权/`
- `05-联调与验收/`
- `06-运维与发布/`
