## 我们在设计什么系统

MyCC 是一个 `Claude + Codex` 协同的智能体系统，目标是把“聊天、任务执行、记忆沉淀、技能治理”统一到一个可追溯、可恢复的工程底座中。

系统边界：
1. 前端：统一聊天入口（PC/移动端）与任务可视化。
2. 后端：任务编排（PLAN/EXECUTE/EVAL/REFLECT）、模型路由、SSE 事件流。
3. 记忆：文件系统 + SQLite 分层记忆。
4. 技能：基于 `skills-core` registry 的标准化路由与治理。
5. 联网：执行前 EvoMap 经验预检（只读）。

当前非目标：
1. 不做重型微服务拆分。
2. 不默认开启 tunnel。
3. 不追求一次性全平台原生客户端。

- `00-基线/`：技术栈冻结、工程约束、版本策略

- `02-契约/`：API 契约、事件字典、错误码规范
- `03-路线图/`：分期计划、任务拆解、里程碑
- `04-评审优化/`：评审结论、优化记录、待拍板事项
- `05-前端体验/`：信息架构、交互规范、页面实现约束
- `06-安全与鉴权/`：配对协议、限流与审计规范
- `07-联调与验收/`：SSE 可靠性与 replay 联调规范
- `08-运维与发布/`：迁移、备份恢复、发布升级策略
- `09-开发准备/`：开工清单、蜂巢并行协议、Agent 设计输入模板

## MVP 最小闭环验收

1. 创建任务成功并生成 `task_id/run_id`。
2. 阶段事件可持续推送（SSE）。
3. 事件先落库后推流，断线后可按 `seq` 回放。
4. 任务产物可落盘并带 `manifest/hash`。
5. 刷新或重启后可恢复任务快照。
